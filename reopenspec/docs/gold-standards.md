# Gold standard files — ReOpenSpec

Generated by `/reo-blueprint` (brownfield pass). Use these as structural references for new code.

## Gold standard files

### CLI / library (TypeScript)

1. **`src/lib/reopenspec-config.ts`**
   - **Score:** 9/10
   - **Why selected:** Clear config loading, defaults, path resolution, and file IO boundary for `reopenspec.json`.
   - **Patterns:** Small exported functions, explicit types (`ReopenSpecConfigFile`, `ResolvedReopenSpecConfig`), fail-soft reads with useful paths in errors.

2. **`src/lib/baseline.ts`**
   - **Score:** 8/10
   - **Why selected:** Central orchestration for scanner behavior — language profile, parser dispatch, baseline shape.
   - **Patterns:** Async `buildBaseline`, composes `detect-profile` and parsers; keeps oclif out of core logic.

3. **`src/commands/init.ts`**
   - **Score:** 8/10
   - **Why selected:** Representative oclif command — flags, user prompts (`readline`), composition of inject, baseline, rules, MCP setup.
   - **Patterns:** `@oclif/core` `Command`, `Flags`, incremental logging, try/catch around optional steps.

### Frontend

_Not applicable as a large UI codebase._ For the VS Code extension, see **`editors/vscode/src/extension.ts`** as the integration entry (thin CLI wrapper; not ranked as application UI).

### Testing
